优化详情

2.1 策略概述

Genre Anchor 策略为每种网文类型定义 4 个维度的约束：类型定义 (Genre Definition)、核心套路 (Core Tropes)、禁用模式 (Forbidden Patterns)、设定规则 (Setting Rules)。以下为仙侠 (xianxia) 类型的完整 Prompt 模板：

## ===== Genre Anchor: xianxia ===== Genre Definition: 仙侠/修真小说以中国传统道教文化为基底，核心叙事围绕"修炼—突破—飞升"的成长体系展开。世界观包含灵气、丹田、金丹、元婴、元神等修真境界设定。 Core Tropes: - 修炼体系：炼气→筑基→金丹→元婴→化神→渡劫→飞升 - 核心冲突：正邪之争、资源争夺、天道因果 - 常见场景：宗门试炼、秘境探险、丹药炼制、法宝祭炼 Forbidden Patterns: - 禁止混入西方奇幻元素（如魔法、精灵、龙骑士） - 避免使用现代科技概念（如基因改造、量子力学） - 不得使用非中国传统文化命名（如希腊神话角色） - 修炼突破必须遵循境界递进顺序，不可跳级 Setting Rules: - 战斗描述需基于灵力/法宝/阵法，而非物理拳脚 - 时间跨度可以极大（百年修炼为常态） - 因果报应设定必须贯穿整个世界观 - 宗门/家族为社会基本组织单位

关键发现

  • 历史类提升最大：faithfulness 从 0.31 跃升至 0.48 (+0.17)，时代锚定规则有效减少了跨时代概念混淆。但 0.48 仍属较低水平，需进一步优化。
  • 科幻类仍有差距：faithfulness 0.52 虽较基线提升 0.10，但技术名词幻觉问题未根本解决。Agent 在硬科幻场景中仍会创造不存在的"伪科技"概念。
  • 轻小说类提升最小：faithfulness 仅 +0.06，因其基线已相对较高，边际收益递减。
  • 任务成功率：所有类型均有提升，历史 (+7.3%) 和悬疑 (+5.3%) 增幅最大，表明 Genre Anchor 对复杂类型的效果最为显著。
4

剩余问题与改进方向

仍存在问题
  • 历史类 faithfulness 0.48 — 虽大幅提升，但距离可用标准 (0.70+) 仍有差距。Agent 在朝代内部细节（官制、服饰、礼节）上仍频繁出错。
  • 科幻类 faithfulness 0.52 — 技术原理描述的幻觉率仍高，"伪科学"名词生成问题突出。
  • 上下文召回 0.59 — 长上下文场景的早期信息丢失问题未根本解决，且受 Prompt 增长影响，token 消耗有所增加。
  • Prompt 长度增加约 800 tokens，轻微推高了响应延迟。
下轮优化方向
  • 生成参数调优：调整 temperature、top_p 等参数以平衡忠实度与创造性。
  • 上下文窗口管理：引入滑动窗口与智能截断策略，解决长上下文退化问题。
  • 历史类专项增强：增加历史时期-技术映射表作为参考知识。
  • 科幻类知识注入：为常见科幻概念添加技术原理锚定定义。

ROUND 02 小结

Genre Anchor 策略证明了 Prompt 工程对 Agent 类型感知能力的显著提升效果。8 项核心指标全线正向增长，其中 faithfulness 提升 13.8%，历史类提升幅度达 54.8%。然而，历史 (0.48) 与科幻 (0.52) 类型仍是薄弱环节，需要在生成参数与上下文管理方面进一步优化。同时应注意 Prompt 长度增加带来的效率损耗，下一轮需在保持质量的前提下控制 Prompt 规模。
