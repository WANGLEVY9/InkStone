本轮优化概要

ROUND 11 将优化焦点从情节/结构转向角色一致性。我们构建了"角色档案库"系统，为每个主要角色维护结构化角色卡，涵盖人格特质、说话风格、知识边界和人际关系。角色卡直接嵌入 prompt 模板，使 Agent 在续写时始终保持对各角色设定的精确记忆。

  • 角色档案库 — 23 个角色档案卡，覆盖 8 类 Genre 中的主要角色类型
  • 对话风格矩阵 — 每个角色的用语习惯、语气特征、口头禅等标注
  • 知识边界约束 — 角色只能知道其设定范围内应知的信息
  • 新增评估维度 — character_consistency 数据集 (290+ 评估单元)

残差分析

  • 配角一致性偏低 — 档案库仅覆盖主要角色，次要角色 OOC 率仍达 15%
  • 跨章角色成长缺失 — 角色卡是静态的，不支持角色随时间成长变化
  • 历史 Genre 挑战大 — 李慕白角色一致性 0.79，因历史语境对言行约束极严格
  • Prompt 长度膨胀 — 23 个角色档案约占用 1800 tokens，压缩空间有限
