本轮优化概要

ROUND 09 聚焦于负面样本对抗训练。通过分析前八轮评估中积累的 200+ 失败案例，我们归纳出 12 类典型错误模式，并将"错误类型检测清单"直接嵌入系统提示词中。同时引入自一致性检查机制：Agent 在生成续写内容后，自动对自身输出进行一轮错误模式扫描，若命中已知错误类型则触发修正流程。

  • 错误类型检测清单 — 12 种错误模式，每种附带检测规则和修正建议
  • 自一致性检查 — 输出后处理环节，Agent 自我审查并修正潜在错误
  • 动态权重调整 — 根据错误类型的严重程度动态调整惩罚权重

自一致性检查机制

自一致性检查 Prompt 示例

[自一致性检查] 请对你刚刚生成的续写内容进行以下检查： 1. [E01] 检查所有角色名称：是否有将角色A的行为归因于角色B？ 2. [E02] 检查时间线：事件顺序是否与原文一致？ 3. [E03] 检查世界观设定：是否有违背核心规则之处？ 4. [E04] 检查逻辑因果：是否有自相矛盾或因果断裂？ 5. [E05] 检查人称代词：是否有不一致之处？ 6. [E08] 检查重复性：是否有与前文重复的内容？ 如果发现上述任何问题，请定位具体位置并给出修正版本。 若全部通过，回复 "[一致性检查通过]"

自一致性检查在 Agent 生成续写后立即执行，定位到错误后触发修正流程。修正后的内容会再次通过检查直至通过或达到最大重试次数（当前设置为 3 次）。

Trade-off 分析

⚠️ 负面影响

  • Token 消耗增加约 8%（自一致性检查 + 修正迭代）
  • 响应时间从 1420ms 增至 1680ms（+18%）
  • 误报率约 7%（将正确内容误判为错误）
  • 复杂场景下修正迭代可能超过 3 次上限

✅ 正面收益

  • 负样本检测率提升 23 个百分点（45% → 68%）
  • Faithfulness 从 0.78 提升至 0.80
  • 修正后的错误中 62% 成功恢复
  • 为后续角色一致性和伏笔追踪奠定基础
